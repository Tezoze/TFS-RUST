# Engine Architecture & Implementation Lessons

1. **Legacy C++ Parity**: Always refer back to the exact legacy TFS C++ source code when answering questions, reproducing behaviors, or making architectural decisions for edge cases. Do not assume behavior; find the correct `.cpp` file using `grep_search` in the legacy `src/` directory and mirror its logic. 
*(Added: April 2026 after spawning parser error logic mismatch)*

2. **TFS RSA is raw modular exponentiation, not PKCS#1 decrypt**: The `rsa` crate‚Äôs `rsa::hazmat::rsa_decrypt` applies blinding and PKCS#1-style semantics for standard encrypted messages. TFS (`src/rsa.cpp`) uses CryptoPP `CalculateInverse` ‚Äî effectively **c^d mod n** on the 128-byte integer with **no** PKCS#1 unpadding. Those two paths are not equivalent; using the hazmat path against OTClient ciphertext can decrypt to garbage even with the correct key. The fix is **`BigUint::modpow(&d, &n)`** (or equivalent) and then fixed-width 128-byte big-endian encoding of the result, matching C++ `Integer::Encode`. See `crates/tfs-rust-net/src/rsa.rs`.  
*(Added: April 2026 ‚Äî handshake ‚Äúwrong key‚Äù / PKCS#1 first-byte errors)*

3. **Login RSA offset depends on client protocol version and OTClient branch**: Do not assume a single ‚Äúlegacy‚Äù layout. For example **protocol 1098 / OTClient v8** `ProtocolLogin:sendLoginPacket` prepends **`ClientEnterAccount` (u8 opcode 0x01)** and fields such as **u32 client version**, **u16 content revision**, padding, **spr/pic signatures**, and **u8 preview state** before the RSA block ‚Äî so the ciphertext starts at **offset 26** from the start of the frame body **after** the 4-byte Adler checksum prefix (i.e. `body[26..158]` when `body[0..4]` is checksum). Older layouts use **15** (game), **25** (login ‚â•971), or **20** (login before 971). When in doubt, read the **actual client source** (e.g. OTClient v8 Lua) rather than guessing offsets. See `crates/tfs-rust-net/src/game_first_packet.rs`.  
*(Added: April 2026 ‚Äî RSA decrypt OK only after fixing offset for 1098)*

4. **Shutdown save vs logout save**: On normal disconnect, `PlayerSaveData` is built on the `LocalSet` game thread and passed into `tokio::spawn` (I/O off the sim loop; DB is `Send`). On **SIGINT**, the same `build_player_save_data` + `PlayerStore::save_player` must **finish before process exit** ‚Äî use a dedicated `GameCommand::Shutdown` branch and **`await`** a bounded work queue (`JoinSet` with a max in-flight cap of 8 matches the plan‚Äôs `buffer_unordered(8)`). Do not block the game port `accept` loop on the main `LocalSet` future: `spawn` both listeners so `await`ing the `spawn_local` game task can run to completion, then `abort` the acceptors. *(`tasks/config-and-save-plan.md` P4, April 2026)*

5. **Config.lua vs environment (network + DB)**: `NetConfig` / `DbConfig` in `config.rs` mirror TFS C++ `configmanager.cpp` keys and defaults. For containers and one-off tests, **env still overrides**: `DATABASE_URL` replaces the `mysql://‚Ä¶` string built from `mysqlHost` / `mysqlUser` / ‚Ä¶; `TFS_LOGIN_ADDR` and `TFS_GAME_ADDR` override bind addresses; `TFS_GAME_PORT` and `TFS_PUBLIC_IP` override what clients are told for the game server. If unset, values come from `config.lua` (and `TFS_CONFIG` chooses which file to load). *(`tasks/config-and-save-plan.md` P1‚ÄìP2 + P5, April 2026)*

6. **Password hashing (bcrypt + legacy SHA1)**: Rust auth deviates from TFS 1.4.2 plain SHA1. Stored format is self-describing: 40 lowercase hex = legacy SHA1; `$2a$`/`$2b$`/`$2y$` = bcrypt. Successful SHA1 login upgrades to bcrypt on the spot. bcrypt verify/hash runs on `tokio::task::spawn_blocking` (I/O path, not game thread). `$2b$` hashes verify under PHP `password_verify`. Set `legacySha1Enabled = false` in `config.lua` once migration is complete. **Startup runs SQLx migrations** (`run_migrations` in `run_server.rs`). For existing C++ TFS databases, `heal_and_adopt_existing_schema` clears failed `_sqlx_migrations` rows and marks the baseline applied without re-running DDL; only incremental Rust migrations (e.g. password widen) execute. Baseline seed uses `INSERT IGNORE` into `server_config`. *(`crates/tfs-rust-db/src/migrations.rs`, May 2026)*
